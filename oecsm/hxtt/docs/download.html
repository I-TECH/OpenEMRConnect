HXTT Access Packages Downloads

The current version of the HXTT Access packages are available here:

  • Because there are several versions for each JDBC release (JDBC1.2, JDBC2.0, JDBC3.0, JDBC4.0, and JDBC4.1), please check that you download the correct build for your VM (or the destination VM).
  • The JDBC1.2 Package is intended for JDK1.1.X and Personal Java environments but will run under later JDKs (1.2.X, 1.3.X, 1.4.X, 1.5.X, 1.6.X, and 1.7.X)
  • The JDBC2.0 Package is intended for JDK 1.2.X and JDK 1.3.X environments but will run under later JDKs (1.4.X, 1.5.X, 1.6.X, and 1.7.X). It will not run under JDK 1.1.X.
  • The JDBC3.0 Package is intended for JDK 1.4.X environments but will run under later JDKs (JDK1.5.X, 1.6.X, and 1.7.X). It will not run under JDK 1.1.X, 1.2.X, or 1.3.X.
  • The JDBC4.0 Package is intended for JDK 1.6.X environments. It will not run under JDK 1.1.X, 1.2.X, 1.3.X, 1.4.X, or 1.5.X.
  • The JDBC4.1 Package is intended for JDK 1.7.X environments. It will not run under JDK 1.1.X, 1.2.X, 1.3.X, 1.4.X, 1.5.X, or 1.6.X.
  • The demo package includes some sample code, and it also has several versions for each JDBC release (JDBC1.2, JDBC2.0, JDBC3.0, JDBC4.0, and JDBC4.1).
  • The Remote Access package is the client side package for client/server mode, and it also has several versions for each JDBC release (JDBC1.2, JDBC2.0, JDBC3.0, JDBC4.0, and JDBC4.1).
  • The Embedded package is one lighweight package without client/server mode support, and it also has several versions for each JDBC release (JDBC1.2, JDBC2.0, JDBC3.0, JDBC4.0, and JDBC4.1).
  • JDBC2.0 package includes support for javax.sql.*, which is in JDK 1.3 only. For JDK1.2.X, you need to download the jdbc2_0-stdext.jar from Sun's site if you wish to use those extended functions.
  • JDBC1.2 package's Database GUI Manager requires swing1.0.3, which is available at JFC 1.1 with Swing 1.0.3.
  • All .jar files are compatible with all platforms.
